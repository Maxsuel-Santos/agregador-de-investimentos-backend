# Security Policy

## Supported Versions

Atualmente, fornecemos atualizações de segurança apenas para a versão estável mais recente e a versão LTS (Long Term Support). Como lidamos com dados financeiros, recomendamos sempre utilizar a última versão disponível.

| Version | Supported          |
| ------- | ------------------ |
| 1.1.x   | :white_check_mark: |
| 1.0.x   | :x:                |
| < 1.0   | :x:                |

## Reporting a Vulnerability

A segurança dos dados financeiros e das integrações de nossos usuários é nossa prioridade máxima. Se você encontrar uma vulnerabilidade, agradecemos imensamente o envio de um relatório privado.

### How to Report

**Por favor, não abra uma "Issue" pública para reportar vulnerabilidades de segurança.** Para garantir a proteção dos usuários, siga um destes canais:

1.  **GitHub Security Advisory:** Vá até a aba "Security" deste repositório e selecione "Report a vulnerability".
2.  **E-mail:** Envie um relatório detalhado para `security@finncker.com`.

**No seu relatório, inclua:**
* Uma descrição clara da vulnerabilidade e o impacto potencial.
* Passos detalhados para reproduzir o problema (PoC).
* Seu nome ou handle para créditos (opcional).

### What to Expect

* **Confirmação:** Você receberá uma resposta confirmando o recebimento em até **48 horas**.
* **Correção:** Nossa equipe trabalhará na triagem e correção. Manteremos você informado sobre o progresso a cada **3-5 dias úteis**.
* **Divulgação:** Após a correção e o lançamento da nova versão, publicaremos um comunicado de segurança dando os devidos créditos aos pesquisadores envolvidos.

### Responsibilities

Pedimos que você siga os princípios de **divulgação responsável**:
* Não tente acessar, modificar ou extrair dados de outros usuários.
* Não utilize a vulnerabilidade para interromper o serviço (DoS).
* Dê-nos um tempo razoável para corrigir o problema antes de qualquer divulgação pública.
